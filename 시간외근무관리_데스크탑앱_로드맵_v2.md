- **버전**: v2.1
- **목표**: 개인용 시간외근무관리 웹앱 → Electron 데스크탑 앱 전환

## 📖 배경 및 목적

### 1. 현재 상태 (개인용 웹앱)

```
[시간외근무관리.html] - 단일 HTML 파일 (11,710줄, 177개 함수)
    │
    ├── 데이터 저장 → 브라우저 localStorage
    ├── 직원 데이터 → 인사관리 웹앱의 localStorage에서 읽기
    ├── 코드 보호 → 없음 ⚠️
    └── 라이선스 → 없음 ⚠️
```

### 2. 목표 상태 (데스크탑 앱)

```
[시간외근무관리 Desktop App - Electron]
    │
    ├── 코드 보호 → Electron 패키징 (F12 비활성화) ✅
    ├── 라이선스 → 인사관리 앱 라이선스 검증
    ├── 직원 데이터 → 인사관리 앱 로컬 파일에서 읽기 ✅
    ├── 시간외/근태 데이터 → 자체 로컬 파일 저장 ✅
    └── 자동 업데이트 → GitHub Release 기반
```

### 3. 핵심 원칙

| 원칙 | 설명 |
|------|------|
| 독립 설치 | 인사관리 앱과 별도 설치 파일 (별도 GitHub 저장소) |
| 인사앱 의존 | 인사관리 앱 설치 + 유효 라이선스 필수 |
| 단방향 참조 | 인사앱 데이터 → 읽기 전용 (쓰기 불가) |
| 로컬 저장 | 시간외/근태 데이터는 자체 로컬 파일에만 저장 |
| 서버 미동기화 | 시간외 데이터는 서버에 올리지 않음 |

## ⚠️ 주요 기술 포인트 및 교훈

### 1. localStorage → electron-store (비동기 전환)

```javascript
// ❌ Before: 동기 방식
const data = JSON.parse(localStorage.getItem('key') || '{}');

// ✅ After: 비동기 방식
const data = await OvertimeDB.getOvertimeDaily();
```

**주의**: 기존 동기 함수를 `async`로 전환 시 **호출하는 쪽도 모두 `await` 추가 필요**. Phase 6에서 `renderOvertimeTypeSettings()`가 `loadOvertimeTypeSettings()`를 await 없이 호출한 것이 발견됨.

### 2. 급여 캐시 전략

```javascript
// 초기화 시 1회 전체 로드 → _salaryCache에 저장
await initSalaryCache();

// 이후 동기 접근 가능
const salary = SalaryCalculator.calculate(employee, year, month);
```

- 인사앱 급여 데이터를 매번 async 호출하면 성능 저하
- 초기화 시 `_salaryCache`에 모든 급여표, 직책수당, 설정 로드
- 이후 `SalaryCalculator`는 동기 함수로 유지

### 3. HTML 구조 검증 중요성

**교훈**: 모듈 추출 시 `<div>` 열기/닫기 **불균형은 치명적**
- 하나의 `</div>` 누락으로 후속 탭이 모두 중첩되어 보이지 않음
- **검증 방법**: 각 탭 영역별 `<div>` / `</div>` 개수 비교

```bash
# 검증 명령 (원본 라인 범위 지정)
opens=$(sed -n "${start},${end}p" file.html | grep -c '<div')
closes=$(sed -n "${start},${end}p" file.html | grep -c '</div>')
```

### 4. CSS 클래스명 일관성

**교훈**: HTML과 CSS의 클래스명이 불일치하면 전체 UI가 깨짐
- HTML에서 `class="app-header"` → CSS에 `.app-header` 정의 필요
- 원본 CSS를 그대로 사용할 때는 **HTML도 원본 클래스명 유지**

### 5. 전역 변수 선언

**교훈**: 원본의 함수 스코프/클로저 내 변수를 모듈로 분리할 때 **전역 변수 선언 필수**
- 원본: `let bulkSelectedRows = new Set();` (클로저 내부)
- 모듈: 파일 최상위에 동일 변수 선언 필요
